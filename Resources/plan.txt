# Your Personal Roadmap: Red Team + VR
## Starting Point: 3 months experience, strong web/API, no network/AD, 1-2hrs/day

---

## The Big Picture First

You have **4 phases** over roughly **3 years**. Red Team and VR run parallel but VR starts slow and builds over time. Think of it like this:

- **Red Team** = your primary income path, builds faster
- **VR** = your long-term wealth path, builds slower but compounds massively

Don't try to do both at 100%. It's **70% Red Team, 30% VR** for the first 2 years, then you decide which to push harder.

# PHASE 1 — Fill Critical Gaps
## Timeline: Months 1–6 (1-2 hrs/day)

You have web/API solid. But Red Team requires 4 pillars. You're missing 3 right now.

---

### Pillar 1 — Networking (You Said You're Weak Here)
**Why:** Everything in Red Team runs over a network. AD attacks, C2 traffic, lateral movement — all of it requires you to understand what's happening at the packet level.

**What exactly to learn:**
- TCP/IP model — how packets actually travel
- DNS, HTTP, SMB, LDAP, Kerberos protocols (these 5 are essential for AD attacks)
- Subnetting and routing basics
- How firewalls and proxies work at a traffic level
- Wireshark — reading and analyzing packet captures

**Free Resources:**
- Professor Messer Network+ (YouTube, completely free) — watch at 1.5x speed
- TryHackMe "Pre-Security" path — free, hands-on, covers networking properly
- Wireshark official sample captures — practice reading real traffic
- TCPDump man page + practice in any Linux VM

**Goal by end:** You can look at a Wireshark capture and understand what's happening. You understand what a Kerberos ticket exchange looks like on the wire.

---

### Pillar 2 — Active Directory (The Heart of Red Teaming)
**Why:** 90% of real enterprise environments run Windows AD. If you don't know AD deeply, you are not a Red Teamer — you're a web tester. This is non-negotiable.

**What exactly to learn:**
- What AD is — domains, forests, trusts, OUs
- Authentication protocols — NTLM and Kerberos in depth
- Core AD attacks:
  - AS-REP Roasting
  - Kerberoasting
  - Pass-the-Hash
  - Pass-the-Ticket
  - DCSync
  - BloodHound enumeration
  - GPO abuse
- Basic AD enumeration with PowerView, SharpHound, BloodHound

**Free Resources:**
- TryHackMe "Active Directory Basics" room — free
- TCM Security "Practical Ethical Hacking" (paid but ~$30, worth every rupee) — has full AD section
- HTB Starting Point machines — free tier, several AD focused
- PayloadsAllTheThings GitHub — AD attack cheatsheet, reference it constantly
- BloodHound official documentation — read the whole thing

**Goal by end:** You can take a foothold in an AD environment and enumerate paths to Domain Admin using BloodHound.

---

### Pillar 3 — Linux and Windows Internals Basics
**Why:** You need to understand what you're operating inside — how processes work, how privileges are structured, how shells behave.

**What exactly to learn:**

**Linux:**
- File permissions, SUID/SGID bits
- Process management, /proc filesystem
- Privilege escalation vectors — sudo misconfigs, cron jobs, weak file permissions, capabilities
- Bash scripting for automation

**Windows:**
- Registry structure
- Windows token model and privileges
- UAC and how to bypass it
- Windows services and scheduled tasks as attack vectors
- PowerShell for offensive use

**Free Resources:**
- GTFOBins — Linux privesc reference (bookmark this now)
- LOLBAS — Windows living-off-the-land binaries reference (bookmark this too)
- TryHackMe "Linux PrivEsc" and "Windows PrivEsc" rooms — both free
- HackTricks GitBook — free, covers everything, use as reference constantly

---

### Your Web/API — Don't Drop It, Sharpen It
You're already good here. During Phase 1 just keep doing PortSwigger labs. Specifically finish:
- Business logic vulnerabilities
- Access control
- OAuth and SSO attacks
- HTTP request smuggling
- GraphQL attacks

These are the web skills that matter in real Red Team engagements, not just bug bounty.

# PHASE 2 — Red Team Core Skills
## Timeline: Months 7–14

This is where you start becoming a real Red Teamer.

---

### Command and Control (C2) Frameworks
**What to learn:**
- How C2 infrastructure works conceptually — listeners, beacons, callbacks
- Cobalt Strike concepts (even without a license — learn the theory)
- Havoc C2 — free and open source, actively maintained
- Sliver C2 — free, modern, used in real ops
- How to set up redirectors to hide your C2 infrastructure
- Domain fronting concepts

**Free Resources:**
- Havoc C2 GitHub + documentation
- Sliver C2 GitHub + documentation
- RedTeamNotes.io — free, practical C2 notes
- C2 Matrix (thec2matrix.com) — compares all C2 frameworks

---

### Evasion — The Skill That Separates Average From Elite
**Why:** Any script kiddie can run Mimikatz. The Red Teamer who can run it on a fully patched, EDR-protected system — that person is valuable.

**What to learn:**
- How AV/EDR detects malware — signatures, heuristics, behavioral detection
- AMSI (Antimalware Scan Interface) — what it is and bypass techniques
- ETW (Event Tracing for Windows) — how defenders use it and how to blind it
- Process injection techniques — DLL injection, process hollowing, shellcode injection
- Payload obfuscation — encoding, encryption, polymorphism
- Living off the land — using Windows built-in tools to avoid detection (LOLBAS)

**Free Resources:**
- "Malware Development for Ethical Hackers" — search this on GitHub, free resources
- VX-Underground — malware samples and papers, free
- ired.team — incredible free Red Team notes covering evasion deeply
- S3cur3Th1sSh1t GitHub — evasion research, free

---

### Active Directory — Advanced (Building on Phase 1)
**What to learn:**
- Kerberos delegation attacks — unconstrained, constrained, resource-based constrained delegation
- ACL/ACE abuse
- Cross-domain and cross-forest attacks
- MSSQL attacks in AD environments
- Certificate Services attacks (ESC1-ESC8) — this is currently massively exploited in real engagements
- Shadow Credentials attack

**Free Resources:**
- SpecterOps blog — free, they literally invented half these techniques
- ly4k's Certipy tool + blog — AD CS attacks, free
- HTB Pro Labs "Offshore" and "RastaLabs" — paid but ~$40/month, worth it when ready

---

### Custom Tooling — Start Writing Your Own
**Why:** Top 1% Red Teamers don't just use other people's tools. They write their own because custom tools bypass signature detection and show real skill.

**What to learn:**
- C# for offensive tooling — most Windows offensive tools are written in C#
- Go for Red Team tools — fast compilation, cross-platform, great for C2 implants
- Python for automation and quick scripts (you already know this)
- Win32 API — how to call Windows APIs directly from code

**Free Resources:**
- "Offensive C#" — search GitHub, multiple free repos
- Golang official tour (go.dev/tour) — free, learn Go basics in a weekend
- OffensiveC# GitHub by matterpreter — free examples

# PHASE 3 — VR Track Starts Seriously
## Timeline: Months 7–18 (parallel to Phase 2, 30% of your time)

---

### C Programming Foundations
**What exactly to learn:**
- Pointers and memory management
- Stack vs heap
- How malloc/free works
- Buffer operations — strcpy, memcpy, why they're dangerous
- Compiling and debugging with GCC and GDB

**Free Resources:**
- CS50 (Harvard) — free on edX, excellent C foundation
- "Beej's Guide to C Programming" — free online, clear and practical
- GDB tutorial on ired.team — free

---

### x86/x64 Assembly — Reading Level Only
You don't need to write assembly. You need to read it.

**What to learn:**
- Registers — what RAX, RBX, RSP, RBP, RIP do
- Basic instructions — MOV, PUSH, POP, CALL, RET, JMP, CMP
- How function calls work at assembly level — prologue and epilogue
- Stack frames — how they're built and destroyed

**Free Resources:**
- pwn.college "Assembly Refresher" — free, interactive
- LiveOverflow YouTube — "How a CPU works" series, free
- x86asm.net reference — free cheatsheet

---

### Classical Binary Exploitation
**What to learn in order:**
1. Stack buffer overflows — 32-bit first, then 64-bit
2. Return-oriented programming (ROP)
3. Format string vulnerabilities
4. Heap basics — use-after-free, double free

**Free Resources:**
- pwn.college — completely free, this is the #1 resource for this
- exploit.education — free VMs (Phoenix, Protostar series)
- CTFtime.org — do pwn category challenges in beginner CTFs
- LiveOverflow "Binary Exploitation" playlist — free on YouTube

---

### Reverse Engineering
**What to learn:**
- Ghidra — static analysis, reading decompiled code
- GDB with pwndbg plugin — dynamic analysis, runtime debugging
- Recognizing common patterns — loops, conditionals, function calls in disassembly
- Analyzing simple crackmes first, then real binaries

**Free Resources:**
- Ghidra — free (NSA released it)
- pwndbg GitHub — free GDB plugin
- crackmes.one — free reverse engineering challenges
- "Ghidra Book" first 3 chapters — free preview available

### Red Team — What Top 1% Do That Others Don't

**Simulate real APT TTPs:**
- Study MITRE ATT&CK framework deeply — not just read it, understand each technique mechanically
- Read real APT reports — Mandiant, CrowdStrike, Microsoft MSTIC publish these free
- Replicate those techniques in your lab
- Build full attack chains — initial access → persistence → lateral movement → exfil

**Build a home lab:**
- 2 Windows Server VMs (one as DC, one as member server)
- 2 Windows 10/11 workstation VMs
- 1 Kali Linux attacker VM
- Install AD, configure users, set up BloodHound
- This costs nothing if you use free VMware/VirtualBox and evaluation Windows ISOs

**Specialize in one area:**
- Initial Access specialist — phishing, macro development, browser exploitation for initial foothold
- Infrastructure specialist — C2 setup, redirectors, OPSEC
- AD specialist — the person who always finds the path to DA
- Evasion specialist — the person who gets past any EDR

**Certs that actually matter for Red Team (in order of value):**
1. CRTO (Red Team Ops by Zero-Point Security) — ~$400, best value Red Team cert
2. OSCP — if you don't have it, get it. Industry baseline.
3. CRTE (Certified Red Team Expert) — AD focused, excellent
4. OSEP — advanced, do this after CRTO

### VR — What Top 1% Do

**Start fuzzing real targets:**
- Pick a small open source project with a GitHub
- Write an AFL++ harness for it
- Run it for a week
- Triage every crash

**Read Project Zero religiously:**
- Every bug writeup they publish — read it fully

**File your first CVE:**
- Target: small-to-medium open source projects

**Free Resources for this phase:**
